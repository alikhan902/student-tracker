## UC-02: Вход в систему

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-02 |
| **Название** | Вход в систему |
| **Акторы** | Гость |
| **Предусловие** | Пользователь зарегистрирован |
| **Постусловие** | Выдан JWT, доступ к `/api/**` |
| **Триггер** | Отправка формы входа |

### Основной поток

| Шаг | Актор | Действие |
|-----|-------|---------|
| 1 | Пользователь | Открывает `/login` (web) или экран входа (desktop) |
| 2 | Пользователь | Вводит username и password |
| 3 | Система (C) | `AuthController` → `POST /auth/login` |
| 4 | Система (M) | `AuthService` через `AuthenticationManager` |
| 5 | Система (F) | `UserRepository.findByUsername` |
| 6 | Система (M) | Генерация JWT (`AuthUtil`) |
| 7 | Клиент | Сохраняет `accessToken` (localStorage / состояние desktop) |

### Альтернативный поток A1: Неверный пароль

`BadCredentialsException` → HTTP 401 → сообщение на UI.

### Альтернативный поток A2: OAuth2

Редирект на Google/GitHub → `OAuth2successHandler` → редирект на `/oauth2/redirect?token=...`.

## UC-09: Создание учебной группы

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-09 |
| **Название** | Создание группы |
| **Акторы** | Староста (HEADMAN) |
| **Предусловие** | Аутентификация; у пользователя ещё нет группы |
| **Постусловие** | Создана запись `Group`, пользователь привязан |
| **Триггер** | `POST /api/groups` |

### Основной поток

| Шаг | Действие |
|-----|---------|
| 1 | `GroupController` принимает `GroupDTO` |
| 2 | `GroupServiceImpl` проверяет `StudentType.HEADMAN` |
| 3 | Сохранение в `GroupRepository` |
| 4 | Возврат DTO группы |

## UC-20: Загрузка образовательного материала

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-20 |
| **Название** | Создание материала |
| **Акторы** | Староста |
| **Предусловие** | Существует `TrainingSubject` группы |
| **Постусловие** | Запись `EducationalMaterial`, файл в `uploads/` |
| **Триггер** | `POST /api/educational-materials` (multipart) |

### Основной поток

| Шаг | Действие |
|-----|---------|
| 1 | Multipart: часть `data` (JSON DTO) + часть `file` |
| 2 | `EducationalMaterialServiceImpl` валидирует предмет |
| 3 | `FileServiceImpl` сохраняет файл |
| 4 | JPA сохраняет сущность, возвращает DTO |

## UC-24: Отметка выполнения материала

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-24 |
| **Название** | Обновление статуса выполнения |
| **Акторы** | Студент / староста |
| **Предусловие** | Материал доступен в группе пользователя |
| **Постусловие** | Upsert `MaterialComplete` |
| **Триггер** | `PUT /api/material-complete/{materialId}` |

### Основной поток

| Шаг | Действие |
|-----|---------|
| 1 | `MaterialCompleteController` принимает `MaterialCompleteUpdateDTO` |
| 2 | `MaterialCompleteServiceImpl` находит или создаёт запись по `(userId, materialId)` |
| 3 | Устанавливает `CompletionStatus` |
| 4 | Возврат актуального статуса |
