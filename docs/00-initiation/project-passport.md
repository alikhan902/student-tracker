## 2. БИЗНЕС-КОНТЕКСТ

**Проблема:**  
Студенческие группы ведут учебные материалы в разрозненных каналах (мессенджеры, облачные папки, таблицы). Старосте сложно централизовать предметы и файлы, а студентам — фиксировать, что уже изучено.

**Решение:**  
Student Tracker предоставляет единый сервис: староста создаёт группу, добавляет предметы и загружает материалы; студенты отмечают выполнение. Доступ через веб-браузер или десктопное JavaFX-приложение на общем REST API.

---

## 3. ЦЕЛИ ПРОЕКТА

- Централизовать учебные предметы и материалы в рамках учебной группы
- Разделить роли **староста (HEADMAN)** и **студент (STANDARD)**
- Реализовать REST API с JWT и OAuth2 (Google, GitHub)
- Поддержать веб- и десктоп-клиенты на единой бизнес-логике
- Обеспечить развёртывание через Docker Compose и HTTPS (Nginx)

---

## 5. КЛЮЧЕВЫЕ РИСКИ

| Риск | Вероятность | Стратегия реагирования |
|------|-------------|----------------------|
| Сложность JWT + OAuth2 | Средняя | Spring Security, JJWT, единый success handler |
| Загрузка файлов (multipart) | Средняя | `FileService`, каталог `uploads/`, прокси через Nginx |
| Синхронизация web/desktop | Низкая | Единый REST API как источник истины |
| Настройка SSL | Низкая | Шаблон `nginx/nginx.conf`, mkcert для локали |

## 6. ЗАИНТЕРЕСОВАННЫЕ СТОРОНЫ

| Стейкхолдер | Роль | Интерес |
|-------------|------|---------|
| Староста (HEADMAN) | Администратор группы | Управление группой, предметами, материалами |
| Студент (STANDARD) | Участник группы | Просмотр материалов, отметка выполнения |
| Преподаватель | Оценщик | Соответствие МУ, качество кода и документации |
| Аудитор (AUDITOR) | Наблюдатель | Только чтение (зарезервировано в security) |
