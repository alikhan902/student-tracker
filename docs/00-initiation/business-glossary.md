# Бизнес-глоссарий

| № | Термин | Определение |
|---|--------|------------|
| 1 | **Учебная группа (Group)** | Организационная единица, объединяющая старосту и студентов одного потока |
| 2 | **Староста (HEADMAN)** | Студент с типом `StudentType.HEADMAN`, создающий группу и управляющий предметами |
| 3 | **Студент (STANDARD)** | Участник группы, просматривающий материалы и отмечающий выполнение |
| 4 | **Учебный предмет (Training Subject)** | Дисциплина в рамках группы с названием, описанием и обложкой |
| 5 | **Образовательный материал (Educational Material)** | Файл или запись (лекция, задание), привязанная к предмету |
| 6 | **Статус выполнения (Material Complete)** | Отметка студента: материал изучен (`COMPLETED`) или нет (`NOT_COMPLETED`) |
| 7 | **Роль в системе (RoleType)** | `ADMIN`, `USER`, `AUDITOR` — уровень доступа в Spring Security |
| 8 | **JWT** | JSON Web Token для stateless-аутентификации web и desktop клиентов |
| 9 | **OAuth2** | Вход через Google или GitHub с выдачей JWT на фронтенд |
| 10 | **REST API** | HTTP + JSON (+ multipart для загрузки файлов) |
| 11 | **Сброс пароля** | Одноразовый токен на email с ограничением частоты запросов |
| 12 | **Контейнеризация** | Docker Compose: postgres, backend, frontend, nginx |
