## 2. НАЗНАЧЕНИЕ И ЦЕЛИ

**Назначение:** автоматизация хранения учебных материалов и учёта их изучения внутри студенческой группы.

**Цели:**
1. Веб-интерфейс для студентов и старост
2. Десктоп-клиент как альтернатива
3. Безопасная аутентификация (JWT, OAuth2)
4. REST API для единой бизнес-логики

---

## 3. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 3.1. Аутентификация
- **ФТ-01:** регистрация (логин, пароль, тип студента)
- **ФТ-02:** вход по паролю
- **ФТ-03:** OAuth2 (Google, GitHub)
- **ФТ-04:** сброс пароля по email
- **ФТ-05:** JWT для `/api/**`

### 3.2. Группа
- **ФТ-06:** староста создаёт/редактирует/удаляет группу
- **ФТ-07:** добавление/удаление студентов по логину

### 3.3. Предметы и материалы
- **ФТ-08:** CRUD учебных предметов (multipart, фото)
- **ФТ-09:** CRUD материалов (multipart, файл)
- **ФТ-10:** скачивание файлов через `/uploads/`

### 3.4. Выполнение
- **ФТ-11:** отметка COMPLETED / NOT_COMPLETED per user per material
- **ФТ-12:** просмотр своего статуса

### 3.5. Профиль
- **ФТ-13:** просмотр/редактирование профиля, смена пароля, удаление аккаунта

## 4. API

| Метод | Путь | Описание |
|-------|------|----------|
| POST | /auth/login | Вход |
| POST | /auth/signup | Регистрация |
| GET | /api/users/profile | Профиль |
| PUT | /api/users/profile | Обновление профиля |
| PUT | /api/users/change-password | Смена пароля |
| POST | /api/users/forgot-password | Запрос сброса |
| POST | /api/users/reset-password | Сброс по токену |
| DELETE | /api/users/delete | Удаление аккаунта |
| POST | /api/groups | Создать группу |
| GET | /api/groups/my-group | Моя группа |
| PUT | /api/groups | Обновить группу |
| DELETE | /api/groups | Удалить группу |
| PUT | /api/groups/add-student/{username} | Добавить студента |
| DELETE | /api/groups/delete-student/{id} | Удалить студента |
| * | /api/training-subjects/** | CRUD предметов |
| * | /api/educational-materials/** | CRUD материалов |
| PUT/GET | /api/material-complete/{materialId} | Статус выполнения |
