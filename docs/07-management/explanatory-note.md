## 1. Введение

### 1.1. Актуальность

В учебных группах отсутствует единая система для хранения материалов по предметам и учёта их изучения студентами. Материалы распределены по мессенджерам и облачным дискам; староста не имеет централизованного инструмента для публикации и контроля прохождения учебного контента.

### 1.2. Цель работы

Разработать многокомпонентную информационную систему **Student Tracker** с веб- и десктоп-клиентами, единым REST API, контейнеризованным развёртыванием и полным комплектом проектной документации в каталоге `docs/`.

### 1.3. Задачи

1. Провести инициацию проекта: паспорт, SWOT, матрицу стейкхолдеров, бизнес-модель.
2. Сформировать требования, доменную модель, матрицу трассировки.
3. Спроектировать архитектуру PCMEF, БД PostgreSQL, REST API.
4. Реализовать backend (Spring Boot), frontend (React), desktop (JavaFX).
5. Обеспечить тестирование, статический анализ и отчёт о покрытии (JaCoCo).
6. Подготовить руководства пользователя и администратора, пояснительную записку.

## 5. Проектные решения

### 5.1. Архитектура

- **PCMEF** (Presentation — Control — Mediator — Entity — Foundation) на Spring Boot 3.3.
- Клиенты: **React 19** (веб) и **JavaFX 21** (desktop) на общем REST API.
- БД: **PostgreSQL 16**; файлы — каталог `uploads/`.
- Безопасность: **JWT**, **BCrypt**, **OAuth2** (Google, GitHub).
- Развёртывание: **Docker Compose** + **Nginx** (HTTPS).

Подробнее: [ADR](../02-architecture/adr.md), [пакетная диаграмма PCMEF](../02-architecture/pcmef-package-diagram.md), [ER-диаграмма](../03-database/er-diagram.md).

### 5.2. Нефункциональные требования

| ID | Требование | Реализация |
|----|------------|------------|
| НФТ-01 | Архитектура PCMEF | Слои C/M/E/F в `backend/` |
| НФТ-02 | PostgreSQL 16 | `docker-compose.yml`, JPA |
| НФТ-03 | Docker Compose | 4 сервиса: postgres, backend, frontend, nginx |
| НФТ-04 | HTTPS | Nginx + mkcert / Let's Encrypt |
| НФТ-05 | Покрытие тестами > 40% | JaCoCo + Vitest (см. [отчёт](../06-implementation/test-coverage-report.md)) |
| НФТ-06 | OpenAPI | springdoc `/swagger-ui.html` |
