## 1. Введение

В учебных группах отсутствует единая система для предметов, файлов и отметок о прохождении материалов. Student Tracker закрывает эту задачу для старосты и студентов.

## 2. Анализ предметной области

Основные сущности: **группа**, **пользователь** (староста/студент), **учебный предмет**, **материал**, **статус выполнения**. Документированы в [бизнес-глоссарии](../00-initiation/business-glossary.md) и [доменной модели](../01-requirements/domain-model.md).

## 3. Проектные решения

- Архитектура **PCMEF** на Spring Boot
- Клиенты: **React** и **JavaFX** на общем REST API
- БД: **PostgreSQL**, файлы: каталог **uploads/**
- Безопасность: **JWT**, **BCrypt**, **OAuth2**
- Развёртывание: **Docker Compose** + **Nginx**

Подробнее: [ADR](../02-architecture/adr.md).
